# Customer AI - Advanced Features Guide

## Overview

The Customer AI system provides intent analysis, sentiment detection, escalation, and RAG-based responses. The following feature descriptions are reference designs; some modules were removed in the Industry Standards refactor.

**Removed modules (as of Industry Standards refactor):** `engagement_monitor`, `quality_scorer`, `faq_generator`, `customer_notification_handler`. Their logic is either backend-owned (e.g., notifications) or can be reintroduced with explicit wiring. Chat responses may include `proactive_suggestions` and `quality_score` placeholders if generated by the AI; backend owns acting on them.

**Active capabilities:** Intent extraction, sentiment analysis, escalation detection, dedicated `POST /api/v1/escalate` endpoint, industry-specific prompts and escalation rules. See [BUSINESS_LOGIC.md](../../customer_ai/BUSINESS_LOGIC.md) and [customer_ai_api.md](../api/customer_ai_api.md).

## Feature 1: Proactive Engagement AI

### Purpose
Reduce conversation abandonment and increase conversions through intelligent engagement monitoring and recovery.

### How It Works

**1. Abandonment Detection Algorithm:**

The AI analyzes conversation patterns to detect abandonment risk:

```python
Risk Factors:
├── Abandonment phrases ("let me think", "maybe later")  → +0.25 per phrase
├── Long response delay (>5 minutes)                     → +0.30
├── Short responses (<3 words)                           → +0.25
├── Single question without follow-up                    → +0.25
└── Total Risk Score                                     → 0.0 to 1.0

Risk Levels:
- High: Score ≥ 0.7   (Immediate intervention)
- Medium: Score ≥ 0.4 (Monitor closely)
- Low: Score < 0.4    (Continue normally)
```

**2. Recovery Message Generation:**

Context-aware messages based on abandonment signals:

| Signal Detected | Recovery Message Example |
|-----------------|--------------------------|
| "let me think" | "I completely understand you need time to consider. Would it help if I explained more about how this could benefit you?" |
| "thanks anyway" | "Thank you for considering us! Before you go, we have some special offers available. Would you like to hear about them?" |
| "maybe later" | "Absolutely! I'm here whenever you're ready. Just so you know, [business] offers [key benefit]." |
| Long delay | "I'm still here to help! Is there anything specific I can clarify for you?" |

**3. Upsell/Cross-sell Opportunities:**

The AI identifies opportunities by analyzing:
- Buying signal keywords ("interested in", "looking for", "need")
- Product mentions in conversation
- Price inquiries (indicates serious interest)
- Multiple questions about a category

**Opportunity Types:**
- **Upsell**: Suggest premium version (price 1.2x+ higher)
- **Cross-sell**: Suggest complementary products (different category)
- **Bundle**: Suggest popular combinations

### Configuration

```python
# In Customer AI orchestrator or config
engagement_config = {
    'abandonment_threshold': 0.6,        # Risk score to trigger recovery
    'recovery_delay_seconds': 300,       # Wait 5 minutes before recovery
    'upsell_confidence_threshold': 0.6,  # Minimum confidence for upsell
    'max_upsell_attempts': 2             # Max upsell suggestions per conversation
}
```

### Metrics

Track these metrics in your dashboard:

- **Abandonment Rate**: % of conversations with high risk
- **Recovery Success Rate**: % of recovered conversations that convert
- **Upsell Acceptance Rate**: % of upsell suggestions accepted
- **Revenue Impact**: Additional revenue from upsells

### API Usage

The feature activates automatically in `/api/v1/chat` responses:

```json
{
  "response": "AI generated message...",
  "proactive_suggestions": {
    "abandonment_risk": {
      "at_risk": true,
      "risk_level": "medium",
      "indicators": ["let me think", "short_response"],
      "confidence": 0.65
    },
    "recovery_message": "I completely understand you need time...",
    "upsell_opportunity": {
      "has_opportunity": true,
      "opportunity_type": "upsell",
      "suggested_products": ["Premium Package"],
      "confidence": 0.75
    },
    "upsell_message": "I noticed you're interested in this option! Let me tell you about our premium package..."
  }
}
```

**Backend Action:**
- If `abandonment_risk.at_risk` is true and `risk_level` is 'high', consider sending the `recovery_message`
- If `upsell_opportunity.has_opportunity` and `confidence` > 0.6, send `upsell_message`

### Use Cases

**E-commerce:**
- Customer browsing products → AI suggests bundle deals
- Customer about to leave → AI offers discount code
- Customer viewing expensive item → AI suggests payment plans

**Healthcare:**
- Patient inquiring about service → AI suggests comprehensive package
- Patient hesitant → AI highlights benefits and testimonials

**Restaurants:**
- Customer ordering main course → AI suggests dessert/drink
- Customer browsing menu → AI highlights popular combos

## Feature 2: Conversation Quality Scoring

### Purpose
Ensure high-quality AI responses through continuous self-evaluation and automatic quality monitoring.

### How It Works

**Quality Evaluation Algorithm:**

The AI evaluates each response across 10+ dimensions:

```
Quality Factors:
1. Response Length              → Too short (-0.2) | Optimal (+0.1) | Too long (-0.1)
2. Positive Indicators          → Each (+0.05): specific, detailed, helpful
3. Negative Indicators          → Each (-0.15): "not sure", "don't know"
4. Low-Confidence Phrases       → Each (-0.2): "I think", "maybe", "probably"
5. Answers Customer Question    → Yes (+0.15) | No (-0.1)
6. Business-Specific Info       → Included (+0.1)
7. Call-to-Action               → Present (+0.1)
8. Product Mentions             → Relevant (+0.1)
9. Empathy/Friendliness         → Present (+0.05)
10. Generic vs. Personalized    → Generic (-0.15) | Personalized (+0.1)

Final Score: 0.0 to 1.0
Quality Levels:
- Excellent: ≥ 0.8
- Good: 0.6 - 0.79
- Fair: 0.4 - 0.59
- Poor: < 0.4
```

**Automatic Flagging:**

Responses flagged for review if:
- Quality score < 0.5
- Confidence score < 0.5
- 3+ quality issues detected
- Contains negative indicators

### Configuration

```python
quality_config = {
    'quality_threshold': 0.5,            # Minimum acceptable quality
    'auto_flag_enabled': True,           # Auto-flag low quality
    'review_queue_enabled': True,        # Send to review queue
    'alert_on_trend_decline': True,      # Alert if quality declining
    'min_responses_for_trend': 10        # Responses needed for trend analysis
}
```

### Metrics

- **Overall Quality Score**: Average across all responses
- **Quality Distribution**: % excellent, good, fair, poor
- **Responses Flagged**: % needing review
- **Quality Trend**: Improving / Stable / Declining
- **Review Queue Size**: Number pending review

### API Response

```json
{
  "response": "Yes! At Mario's, we have several vegetarian options...",
  "quality_evaluation": {
    "quality_score": 0.87,
    "confidence_score": 0.92,
    "quality_level": "excellent",
    "issues": [],
    "recommendations": [],
    "needs_review": false,
    "word_count": 45
  }
}
```

**Backend Actions:**
- If `needs_review` is true, add to review queue
- Track `quality_score` over time
- Alert if quality_score drops below threshold for multiple messages
- Use for continuous improvement

### Use Cases

- **Quality Assurance**: Automatic QA without manual review
- **Training Data**: Identify high-quality responses for fine-tuning
- **Performance Monitoring**: Track AI improvement over time
- **Team Alerts**: Notify when quality issues arise

## Feature 3: Customer Intent Prediction

### Purpose
Predict customer intent before explicit statement to provide proactive service and identify high-value customers early.

### How It Works

**Intent Detection Signals:**

**Purchase Intent:**
```
Strong Signals (confidence: 0.8+):
- "I want to buy", "I'll take", "how much", "checkout", "purchase"

Medium Signals (confidence: 0.6-0.8):
- "interested in", "looking for", "need", "available", "tell me about"

Weak Signals (confidence: 0.4-0.6):
- "what do you have", "show me", "browsing", "just looking"
```

**Booking Intent:**
```
Strong Signals (confidence: 0.85+):
- "book", "schedule", "appointment", "reserve", "availability"

Medium Signals (confidence: 0.65-0.85):
- "consultation", "meeting", "visit", "come in"

Weak Signals (confidence: 0.45-0.65):
- "hours", "open", "when are you", "location"
```

**Complaint Intent:**
```
Strong Signals (severity: high):
- "disappointed", "frustrated", "terrible", "refund", "complaint"

Medium Signals (severity: medium):
- "problem", "issue", "wrong", "mistake", "not working"

Weak Signals (severity: low):
- "concern", "question about", "confused", "not sure"
```

**High-Value Customer Indicators:**
```
Indicators:
- Bulk keywords: "multiple", "several", "bulk" → +0.25
- Price inquiries (2+): "how much", "cost", "quote" → +0.20
- Business context: "company", "corporate", "office" → +0.20

Value Score ≥ 0.5 = High-Value Customer
```

### Predicted Actions

Based on intent strength, AI recommends:

| Intent | Strength | Predicted Action | Recommended Strategy |
|--------|----------|------------------|----------------------|
| Purchase | Strong | ready_to_purchase | Facilitate payment immediately |
| Purchase | Medium | needs_more_info | Provide detailed benefits |
| Purchase | Weak | exploring_options | Build trust, showcase value |
| Booking | Strong | ready_to_book | Show available slots now |
| Booking | Medium | considering_booking | Explain benefits |
| Complaint | Strong | escalate_now | Immediate business owner alert |

### Use Cases

**Proactive Service:**
- Customer mentions "looking for" → AI proactively suggests products

**Priority Routing:**

## Feature 4: Complaint Severity Classification

### Purpose
Intelligently classify complaint severity to optimize escalation and resource allocation. Not all negative sentiment requires escalation.

### How It Works

**4 Severity Levels:**

**Critical (Immediate Escalation):**
```
Indicators:
- Legal threats: "lawsuit", "lawyer", "sue", "police"
- Fraud claims: "fraud", "scam", "stolen"
- Safety issues: "danger", "threat"
- Extreme urgency: "immediately", "right now", "emergency"

Action: Escalate to business owner immediately
Notification: Email + SMS + Push (all channels)
```

**High (Escalation Recommended):**
```
Indicators:
- Strong negative emotion: "worst", "terrible", "furious", "outraged"
- Refund requests: "refund", "money back", "cancel"
- Churn threats: "never again", "report"
- Sentiment score < -0.6

Action: Escalate to business owner
Notification: Email + Push
```

**Medium (Attempt Resolution):**
```
Indicators:
- Moderate issues: "problem", "issue", "wrong", "mistake"
- Disappointment: "disappointed", "unhappy", "frustrated"
- Sentiment score: -0.6 to -0.4

Action: AI attempts resolution, escalate if unresolved
Notification: Log for review
```

**Low (Standard Handling):**
```
Indicators:
- Minor concerns: "concern", "question", "confused"
- Inquiries about issues
- Sentiment score: > -0.4

Action: Handle with empathy, monitor
Notification: None (routine)
```

**Intelligent Rules:**
- Repeated complaints (2+ in conversation) → Upgrade severity
- High sentiment + medium indicators → High severity
- Context matters: Product questions don't escalate, service failures do

### Context Summary Generation

When escalating, AI generates comprehensive summary:

```

Conversation Duration: 12 minutes
Total Exchanges: 6

Recent Conversation:

Customer Profile:
- Mood: frustrated
- Urgency: high
- Previous issues: 2 similar complaints

Recommended Action: Immediate escalation. Customer is very upset and may churn.
```


### Use Cases

**Smart Resource Allocation:**
- Critical: CEO/owner notified immediately
- High: Manager handles within 1 hour
- Medium: Team member handles same day
- Low: AI handles, human monitors

**Customer Service Optimization:**
- Focus human resources on truly critical cases
- AI handles routine concerns
- Improve response time for urgent issues
- Reduce escalation fatigue

**Example Scenarios:**

## Feature 4: FAQ Auto-Generation

### Purpose
Automatically learn from customer conversations to build and maintain knowledge base, reducing repetitive questions.

### How It Works

**1. Question Pattern Detection:**

```python
Process:
1. Extract all customer questions from conversations
2. Normalize questions (remove filler words, punctuation)
3. Group similar questions using similarity matching
4. Count frequency of each question pattern
5. Identify candidates (min 3 occurrences)

Similarity Algorithm:
- Word-based matching (Jaccard similarity)
- Threshold: 0.7 (70% word overlap)
- Example: "What are your prices?" ≈ "How much does it cost?"
```

**2. Answer Generation:**

Two methods:

**Method A - Extract from Conversations:**
```python
1. Find conversations containing the question
2. Extract AI responses that followed
3. Identify most common/best answer
4. Validate answer quality
5. Return as FAQ answer
```

**Method B - Generate from Business Data:**
```python
1. Categorize question (pricing, hours, location, etc.)
2. Find relevant business data
3. Generate professional answer
4. Format consistently
```

**3. FAQ Categories (10 types):**
- Pricing: "How much...", "What are your prices..."
- Availability: "Do you have...", "Is ... available..."
- Booking: "How do I book...", "Can I schedule..."
- Location: "Where are you...", "How do I find you..."
- Hours: "When are you open...", "What are your hours..."
- Products: "What products...", "What do you offer..."
- Policies: "What's your return policy...", "Do you offer refunds..."
- Payment: "What payment methods...", "Do you accept..."
- Shipping: "Do you deliver...", "How long for delivery..."
- Support: "How can I contact...", "Where can I get help..."

**4. Update Suggestions:**

```python
FAQ Lifecycle Management:
├── Identify new trending questions (high frequency, not in FAQs)
├── Flag outdated FAQs (not asked in 90 days)
├── Suggest answer improvements (asked frequently but low satisfaction)
└── Recommend FAQ reorganization (popular FAQs to top)
```

### Configuration

```python
faq_config = {
    'min_occurrences': 3,               # Min times question must appear
    'similarity_threshold': 0.7,        # Similarity for grouping (0-1)
    'analysis_window_days': 30,         # Look at last N days
    'auto_add_enabled': False,          # Auto-add to knowledge base
    'categories': ['pricing', 'hours', 'location', ...],
    'max_faqs_suggested_per_week': 10   # Limit suggestions
}
```
